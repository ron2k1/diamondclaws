# Autonomous Bullish Signal Monitor — Spec

## What This Is

A background system that watches 52 stock tickers for bullish signals using real market data. When a stock crosses the threshold, 3 AI subagents silently analyze it, a StockVizzy chart is generated, and a single HTML popup report opens in the user's browser. That popup is the **only** user-facing output — no messages, no DMs, no notifications, no chat responses.

## How It Works

```
Poll Loop (every 2 min)
    |
    v
yf.download() — single batch call, 52 tickers, 5d of 5m bars
    |
    v
For each ticker: compute 4 bullish indicators from OHLCV
    |
    v
Score >= 3/4 AND not on cooldown?
    |
    YES
    |
    v
[Silent — user sees nothing until the popup]
    |
    +---> generate_biased_analysis() x3 (all 3 subagents, concurrent)
    +---> stockvizzy() headless chart (concurrent with above)
    |
    v
Build HTML consensus report (chart + 3 agent takes + signal indicators)
    |
    v
webbrowser.open() — single popup, temp file, user closes it and it's gone
```

## Bullish Indicators

All computed locally from OHLCV data using pandas/numpy (already in deps). No extra API calls.

| Indicator | Signal | Math |
|-----------|--------|------|
| RSI (14-period) | Bullish when > 50 | Standard Wilder RSI on close prices |
| MACD Crossover | Bullish when MACD line crosses above signal line | EMA(12) - EMA(26) vs EMA(9) of MACD |
| Volume Spike | Bullish when volume > 1.5x 20-bar avg AND price up > 0.5% | Current bar vs rolling mean |
| MA Crossover | Bullish when SMA(5) crosses above SMA(20) | Simple moving average cross |

**Trigger threshold**: 3 of 4 must agree (configurable).

**Cooldown**: 15 minutes per ticker after triggering (configurable). Prevents repeat popups for the same stock.

## Subagent Behavior

The 3 personas (Bullish Alpha, Value Contrarian, Quant Momentum) act as **silent subagents**:

- They are invoked via `generate_biased_analysis(ticker, persona_id)` which already handles:
  - Multi-provider LLM routing (via `tools/providers.py`)
  - SOUL.md personality loading
  - Cognitive distortion engine
  - Recommendation derivation
- They run concurrently (all 3 fire at the same time)
- Their output is **never shown to the user directly** — it is only consumed by the report generator
- Each subagent produces: analysis text, recommendation (BUY/SELL/HOLD), confidence level, biases used, hallucinations, distortions applied

## StockVizzy Chart

- Called headless: `stockvizzy(ticker, period="1d", interval="5m", save=temp_path, show=False)`
- Generates a 1800x900px dark-theme PNG with price line, volume bars, high/low markers, stats box
- Runs in ThreadPoolExecutor (matplotlib is blocking)
- Runs concurrently with the 3 subagent LLM calls
- Output PNG is base64-encoded and embedded in the HTML report

### Report Behavior

- **Pops up instantly** when the trigger fires (before any delay)
- **One popup per trigger** — not 3, not per-agent, just one combined report
- **Temp file** — saved to OS temp dir, not persisted
- User closes the tab and it's gone
- Also served via API at `GET /api/consensus/{ticker}/report` (most recent report per ticker)

## Polling Details

| Parameter | Default | Env Var | Notes |
|-----------|---------|---------|-------|
| Poll interval | 120s | `POLL_INTERVAL_SECONDS` | How often to check all tickers |
| Signal threshold | 3 | `SIGNAL_THRESHOLD` | How many of 4 indicators must agree |
| Cooldown | 900s (15 min) | `COOLDOWN_SECONDS` | Per-ticker lockout after trigger |
| Enabled | false | `MONITOR_ENABLED` | Opt-in, disabled by default |

### API Rate Friendliness

- `yf.download(tickers, period="5d", interval="5m")` with all 52 tickers in **one batch call**
- 1 API call every 2 minutes = 30 calls/hour (well within yfinance limits)
- Indicator math is pure local pandas/numpy — zero extra API calls
- LLM calls only fire when a trigger happens (rare — 3/4 indicators must agree)

## API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/monitor/status` | GET | Running state, watchlist, cooldowns, last poll results per ticker |
| `/api/monitor/start` | POST | Start the background poll loop |
| `/api/monitor/stop` | POST | Stop the background poll loop |
| `/api/monitor/configure` | POST | Update poll interval, threshold, cooldown, watchlist at runtime |
| `/api/consensus/{ticker}/report` | GET | Serve most recent consensus HTML report |

## What This Does NOT Do

- No Discord messages
- No Telegram messages
- No Gmail
- No chat responses to the user
- No visible agent activity — subagents are completely silent
- No persistent storage of reports (temp files only, plus in-memory pointer for the API)
- No modification to the existing chat/discuss/analyze endpoints
